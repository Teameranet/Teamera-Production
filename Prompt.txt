- **New users (signup):** Create account → proceed immediately to Onboarding Modal → complete profile → stay logged in

### Sign Up Form
- Collect full name — store in DB, display on Profile page and Navbar instead of email

### Social Auth Buttons (Google & Microsoft)

## 7. Project Application Flow

### Applying to a Project
- User selects one position at a time and submits one application per position
- Application form includes: message, optional resume/document upload
- After submitting: show toast notification at bottom — "Your Application has been sent"
- Redirect back to the project page after submission

### Re-application Scenarios
- **User removed by owner:** Old ACCEPTED application deleted; user can reapply (new application = PENDING)
- **User leaves voluntarily:** Same re-application flow as above
- **Current team member:** Apply buttons hidden; "You are already a team member" notice shown

### Notifications
- **Project owner** receives: "New application received for '[ProjectName]' project"
- **Applicant** receives (on acceptance): "Your application for '[ProjectName] — [Position]' has been successfully accepted"

### Dashboard — Application Management

#### Received Tab (project owners only)
- Shows applications submitted by other users for the owner's projects
- Displays: applicant details, skills, position applied for, uploaded document (downloadable)
- Status badges: PENDING, ACCEPTED, REJECTED
- For PENDING: Accept / Reject action buttons
- Owner can view applicant's profile and send them a message

#### Sent Tab (applicants)
- Shows applications the current user has submitted
- Displays correct status in real time

### On Accept
- Applicant automatically added to the specified project's Collaboration Space as a team member with their position name
- Applicant added to their "participating projects" list
- Profile stats updated for both users
- Collaboration Space Team tab updates in real time

# Project Application & Invitation Feature Requirements

## 1. Implement Application Status Check Feature & update Backend & DB
- Before showing the application form, check if the user has already applied to this position
- If they have a pending or accepted application:
- Show the pop message below the "Apply for this position" button if they have a pending or accept application
- Keep the  "Apply for this position" button button size unchanged
-  On pop message give the Link of application (accept/reject)  redircrt application -> send section to see the application  is reject or not
-  Display a clean SaaS-style message

## 2. Project Owner Invitation Flow
When a project owner invites a member (either during project creation or editing):

### For the Invited Member:
- Show a notification message: "You have been invited by the Project Owner for the '[Position Name]' position"
- Display this message below the "Apply for this position" button if the invitation matches the current position
- If the invitation is for a different position, show the message below that position's card instead
- Provide a link to open the CollaborationSpace
- Use a clean SaaS-style design with a consistent, accessible color palette where each color clearly represents its function

## 4. Backend & Database Updates Required
- Add invitation tracking system
- Store invitation details: project ID, invited user ID, position name, invitation status
- Check for matching invitations when displaying position cards
- Update application status logic to handle "quit" state
- Implement notification system for invitation messages

## 5. Position Matching Logic
- When displaying a position card, check if there are any invitations for that specific position
- If invitation matches current position → show message below "Apply for this position" button
- If invitation is for a different position → show message top that specific position card
- Each invitation message should clearly state the position name
