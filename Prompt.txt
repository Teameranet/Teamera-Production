## Phase 4: Project Applications & Invitations

### 4.1 Application Status Check
**Before showing application form:**
- Check if user already applied to this position
- If pending or accepted application:
  - Show message below "Apply for this position" button
  - Include link to view application status (redirect to Dashboard → Sent tab)
  - Keep button size unchanged
  - Use clean SaaS-style message

### 4.2 Re-application Rules
**Users CAN re-apply when:**
- Previous status: REJECTED
- Previous status: QUIT
- Previous status: REMOVED (owner removed them)

**Users CANNOT re-apply when:**
- Current status: PENDING (already applied, waiting)
- Current status: ACCEPTED (already on team)

**Duplicate Application Blocked:**
- User has PENDING application → tries to apply again
- Backend blocks: "You already have a pending application..."
- Error toast (red) → form stays open

**Already Accepted Blocked:**
- User is on team (ACCEPTED) → tries to apply again
- Backend blocks: "You are already accepted..."
- Error toast (red) → form stays open

### 4.3 Applying to a Project
- User selects one position at a time
- Application form: message + optional resume/document upload
- After submit: toast notification "Your Application has been sent"
- Redirect back to project page

### 4.4 Application Management Dashboard

**Received Tab (Project Owners Only):**
- Shows applications from other users for owner's projects
- Display: applicant details, skills, position applied for, uploaded document (downloadable)
- Status badges: PENDING, ACCEPTED, REJECTED
- For PENDING: Accept / Reject buttons
- Owner can view applicant's profile & send message
- When member leaves: display "quit" instead of "Invite"

**Sent Tab (Applicants):**
- Shows applications user submitted
- Display: project owner details, position applied for, uploaded document
- Status badges: PENDING, ACCEPTED, REJECTED
- View Project button + Resume button (if available)
- NO Profile button
- When member leaves: display "quit" instead of "Invite"

### 4.5 On Accept
- Applicant automatically added to project's Collaboration Space as team member with position name
- Applicant added to "participating projects" list
- Profile stats updated for both users
- Collaboration Space Team tab updates in real-time
- Notification sent to applicant: "Your application for '[ProjectName] — [Position]' has been successfully accepted"

### 4.6 Invitation Notifications
- Project owner receives: "New application received for '[ProjectName]' project"
- Applicant receives (on acceptance): "Your application for '[ProjectName] — [Position]' has been successfully accepted"

## Phase 10: Backend & Database (Supabase)

### 10.1 Setup
- After `npm run server`: console shows "Supabase is Connected"
- Fix any connection errors before proceeding

### 10.2 Data Stored in Supabase
- User accounts (full name, email, onboarding details, profile data)
- Projects (creation, edits, deletions, team members)
- Applications (status, uploaded files, position)
- Invitations (project ID, invited user ID, position name, status)
- Messages (per project, per user)
- Files (uploaded to collaboration space)
- Tasks (per project)
- Notifications

### 10.3 API Endpoints (Backend ↔ Frontend)
- Auth: Sign Up, Sign In, Session refresh
- Profile: Get, Update (real-time)
- Projects: Create, Read, Update, Delete
- Applications: Submit, Accept, Reject, Re-apply
- Invitations: Create, Get, Accept, Reject
- Collaboration: Messages, Files, Tasks, Team members
- Notifications: Get, Mark as read

### 10.4 SQL Files Required
- **File 1:** Auth + Profile + Profile Edit
- **File 2:** Project Creation + Application Flow + Invitations
- Do NOT create any .md files

### 10.5 Security
- Row Level Security (RLS) policies on all tables
- Supabase Auth compromised password check enabled (HaveIBeenPwned)

### 10.6 Real-time Updates
- Profile stats update on project create/join
- Dashboard updates on application status change
- Collaboration Space team tab updates on member add/remove
- Notifications delivered in real-time
- Invitations display in real-time on project modal
